# Architecture decision records

Architecture decision records (ADRs) capture load-bearing design
choices for d2b. They complement the Diataxis documentation tree;
for the broader design narrative, see
[docs/explanation/design.md](../explanation/design.md).

| ADR | Status | Date | Summary |

| [0000. Repository layout and Rust bootstrap](0000-repository-layout-and-rust-bootstrap.md) | Accepted | 2026-05-25 | Establishes `packages/` as the Rust workspace home while preserving the v0.4.0 bash CLI as the only user-facing entry point. |
| [0001. Systemd-free VM orchestration](0001-systemd-free-vm-orchestration.md) | Accepted | 2026-05-25 | Daemon-owned VMs produce no per-VM systemd units, with a `/run/d2b/locks/<vm>` single-writer lock and `[pending restart]` drift surface. |
| [0002. Non-root d2bd plus minimal privileged broker](0002-non-root-daemon-and-privileged-broker.md) | Accepted | 2026-05-25 | `d2bd` runs as a non-root user; a small AF_UNIX broker handles only non-delegable kernel/filesystem mutations and writes an append-only root-owned audit log. |
| [0003. Minijail provisioning and sandbox interface](0003-minijail-provisioning-and-sandbox-interface.md) | Accepted | 2026-05-25 | Nix-pinned minijail behind a typed Rust `d2b-sandbox` API; no `ioctl: 1` catch-alls; `requiresStartRoot` only for audited roles. |

| [0015. Daemon-only clean break (v1.0)](0015-daemon-only-clean-break.md) | Accepted | 2026-05-31 | P6 collapses every persistent root surface to `d2bd` + `d2b-priv-broker`; deletes per-VM systemd templates, host singletons (`d2b-{ch-exporter,otel-host-bridge,…}`), the `cli.nix` bash package, the W14c bash fallback bridge, and the `d2b-launcher` polkit rules. | [0017. | [0018. | [0021. Broker-pre-established user namespace for virtiofsd](0021-broker-user-namespace-for-virtiofsd.md) | Accepted | 2026-06-02 | v1.1.1 retires the [ADR 0003](0003-minijail-provisioning-and-sandbox-interface.md) virtiofsd `requiresStartRoot` carve-out. The broker pre-creates a single-entry user namespace via `clone3(CLONE_NEWUSER)` and writes `0 host_uid_for_zero 1` to `uid_map` so virtiofsd runs fake-root inside the namespace while exposing **zero** host capabilities. Extends [ADR 0001](0001-systemd-free-vm-orchestration.md)'s broker-owned privilege quarantine premise to virtiofsd specifically. |
| [0023. Runner-role lifecycle matrix](0023-runner-role-lifecycle-matrix.md) | Accepted (v1.2) | 2026-06-09 | Every new runner role must fill out an 11-field lifecycle matrix (fork model, wait/reap owner, in-NS mount-action, capability bounding set, ambient capability set, seccomp profile, FD lifetime, umask, RLIMIT_NPROC, oom_score_adj, CLONE_INTO_CGROUP) before being added to `nixos-modules/minijail-profiles.nix`. Backfills matrices for the 5 existing roles (cloud-hypervisor, virtiofsd, swtpm, gpu, audio). Motivated by the vx-kernel retrospective finding that fu27 + fu34 should have been caught at v1.1.2 review. |
| [0022. | [0024. In-VM guest config editing, sync, and containment](0024-in-vm-guest-config-sync.md) | Accepted | 2026-06-07 | Adds `d2b.vms.<vm>.guestConfigFile` (a contained guest-editable OS layer) + the `d2b config` sync/diff/approve workflow. Containment is a fail-closed host-eval namespace policy lint (no `microvm.*`/`d2b.*` from the guest file, detected by definition-existence over the real NixOS module set so imports/`builtins.toFile`/`_file`-spoofing are caught — a best-effort lint, not a sound sandbox: an approved file's eval-time host reads and `mkIf`-on-divergent-context cases are bounded by operator review/approve, with a sound restricted-evaluator as deferred future work); the canonical config stays host-side; the transport is a bounded host-initiated SSH copy reusing the per-VM key (no virtiofs, no new socket); review/approve is host-operator-only and never auto-writes the config tree; guest-built store paths are never trusted. **Superseded transport (W16):** the SSH `cat` copy described here was replaced by the typed bounded `ReadGuestFile` read over the authenticated guest-control vsock (ADR 0029); the rest of the containment/review model still holds. |

| [0030.
