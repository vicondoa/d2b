# Architecture decision records

Architecture decision records (ADRs) capture load-bearing design
choices for nixling. They complement the Diataxis documentation tree;
for the broader design narrative, see
[docs/explanation/design.md](../explanation/design.md).

| ADR | Status | Date | Summary |
| --- | --- | --- | --- |
| [0000. Repository layout and Rust bootstrap](0000-repository-layout-and-rust-bootstrap.md) | Accepted | 2026-05-25 | Establishes `packages/` as the Rust workspace home while preserving the v0.4.0 bash CLI as the only user-facing entry point. |
| [0001. Systemd-free VM orchestration](0001-systemd-free-vm-orchestration.md) | Accepted | 2026-05-25 | Daemon-owned VMs produce no per-VM systemd units, with a `/run/nixling/locks/<vm>` single-writer lock and `[pending restart]` drift surface. |
| [0002. Non-root nixlingd plus minimal privileged broker](0002-non-root-daemon-and-privileged-broker.md) | Accepted | 2026-05-25 | `nixlingd` runs as a non-root user; a small AF_UNIX broker handles only non-delegable kernel/filesystem mutations and writes an append-only root-owned audit log. |
| [0003. Minijail provisioning and sandbox interface](0003-minijail-provisioning-and-sandbox-interface.md) | Accepted | 2026-05-25 | Nix-pinned minijail behind a typed Rust `nixling-sandbox` API; no `ioctl: 1` catch-alls; `requiresStartRoot` only for audited roles. |
| [0004. Cloud Hypervisor runner shape](0004-cloud-hypervisor-runner-shape.md) | Accepted | 2026-05-25 | Generate nixling-owned CH argv from evaluated microvm/nixling config; keep declaredRunner as the parity oracle; reserve future W2 crate names. |
| [0005. | [0006. Manifest bundle versioning](0006-manifest-bundle-versioning.md) | Accepted | 2026-05-25 | `bundleVersion` + per-artifact `schemaVersion`; preserve `vms.json` `manifestVersion = 2`; `deny_unknown_fields` on security-sensitive artifacts; private 0640 bundle paths. |

| [0009. Rust toolchain, MSRV, and supply-chain policy](0009-rust-toolchain-msrv-and-supply-chain.md) | Accepted | 2026-05-25 | Pins the W0a Rust toolchain/MSRV and gates the workspace with reproducible formatting, lint, test, deny, and audit checks. |
| [0010. Wire protocol and typed errors](0010-wire-protocol-and-typed-errors.md) | Accepted | 2026-05-26 | Freezes the W2 Unix-socket wire protocol, negotiation rules, typed-error envelope, and audited broker read path. |
| [0011. cgroup v2 delegation and pidfd handoff](0011-cgroup-v2-delegation-and-pidfd-handoff.md) | Accepted | 2026-05-27 | Single `nixling.slice`, 8-step v2 delegation (Phase A privileged setup steps 1-6, Phase B non-root runtime mutation steps 7-8 per the v1.1 update), leaf-only **broker-mediated** `CgroupKill` (v1.1-P10 op — daemon never writes `cgroup.kill` directly), non-root Phase B delegation invariant, and pidfd handoff over SCM_RIGHTS. **v1.1 update**: the v1.0 `PR_SET_CHILD_SUBREAPER` clause is explicitly SUPERSEDED for the SpawnRunner-child population (NEITHER broker NOR daemon claims subreaper for SpawnRunner children per ADR 0018 § "set-booted race-free serialization"). |
| [0012. | [0013. | [0014. W3 `kernel.modules_disabled=1` behavior, module probe order, CH net handoff selection, and runner-shape preflight](0014-w3-modules-devices-runner-shape.md) | Accepted | 2026-06-08 | Documents the four-step kernel-module probe order, the fail-closed posture under `modules_disabled=1`, the `tap-fd` / `persistent-tap` selection rule, and the runner-shape preflight contract owned by W3 scope s4. |
| [0015. Daemon-only clean break (v1.0)](0015-daemon-only-clean-break.md) | Accepted | 2026-05-31 | P6 collapses every persistent root surface to `nixlingd` + `nixling-priv-broker`; deletes per-VM systemd templates, host singletons (`nixling-{ch-exporter,otel-host-bridge,…}`), the `cli.nix` bash package, the W14c bash fallback bridge, and the `nixling-launcher` polkit rules. The `supervisor` option's hard removal + eval-time rejection assertion is SCHEDULED for v1.1-P2 (v1.0 retains the option for backward-compat with consumer flakes pinning pre-v1.0 manifests); see [ADR 0017](0017-no-bash-fallbacks-invariant.md) + [ADR 0018](0018-microvm-nix-removal.md) for the v1.1 follow-through. No v0 → v1 compat window otherwise; supersedes the migration-mode plumbing in [ADR 0007](0007-bash-coexistence-and-migration.md). |
| [0017. No bash fallbacks invariant (v1.1)](0017-no-bash-fallbacks-invariant.md) | Accepted | 2026-05-31 | v1.1-P1 deletes `exec_legacy_passthrough` + `should_fallback_to_legacy` and converts the 6 residual call sites (audit `--strict`/unreachable, keys list/show unavailable, the two passthrough helpers) to typed envelopes per [ADR 0010](0010-wire-protocol-and-typed-errors.md). CI grep gate `tests/no-bash-exec-eval.sh` enforces the invariant. |
| [0018. Removal of the microvm.nix flake dependency (v1.1)](0018-microvm-nix-removal.md) | Accepted | 2026-05-31 | v1.1-P8 → P11 re-homes every `config.microvm.vms.<vm>.config.config.microvm.*` read to `nixling.vms.<vm>.runner.*`, introduces a nixling-owned per-VM NixOS submodule evaluator (`nixos-modules/vm-submodule.nix`), retires the `microvm@*` / `microvm-virtiofsd@*` / `nixling-<vm>-store-sync.service` template surfaces in favour of broker `SpawnRunner` variants, drops the `microvm` flake input, and updates the tagline to drop "on microvm.nix". |
| [0021. Broker-pre-established user namespace for virtiofsd](0021-broker-user-namespace-for-virtiofsd.md) | Accepted | 2026-06-02 | v1.1.1 retires the [ADR 0003](0003-minijail-provisioning-and-sandbox-interface.md) virtiofsd `requiresStartRoot` carve-out. The broker pre-creates a single-entry user namespace via `clone3(CLONE_NEWUSER)` and writes `0 host_uid_for_zero 1` to `uid_map` so virtiofsd runs fake-root inside the namespace while exposing **zero** host capabilities. Extends [ADR 0001](0001-systemd-free-vm-orchestration.md)'s broker-owned privilege quarantine premise to virtiofsd specifically. |
| [0023. Runner-role lifecycle matrix](0023-runner-role-lifecycle-matrix.md) | Accepted (v1.2) | 2026-06-09 | Every new runner role must fill out an 11-field lifecycle matrix (fork model, wait/reap owner, in-NS mount-action, capability bounding set, ambient capability set, seccomp profile, FD lifetime, umask, RLIMIT_NPROC, oom_score_adj, CLONE_INTO_CGROUP) before being added to `nixos-modules/minijail-profiles.nix`. Backfills matrices for the 5 existing roles (cloud-hypervisor, virtiofsd, swtpm, gpu, audio). Motivated by the vx-kernel retrospective finding that fu27 + fu34 should have been caught at v1.1.2 review. |
| [0022. Stabilization-mode releases](0022-stabilization-mode-releases.md) | Accepted | 2026-06-10 | Defines the stabilization-release flavour of SemVer MINOR: no new public surface, no new deferrals (I3), every tracked deferral closes. Documents the grep-based I3 enforcement in `tests/static.sh` + `make pre-tag`, the required pre-tag live-smoke gate (D1), the R1→R2→R3 panel-signoff cadence, and the annotated-only (no GPG) tag-signing decision for v1.2. |
| [0024. In-VM guest config editing, sync, and containment](0024-in-vm-guest-config-sync.md) | Accepted | 2026-06-07 | Adds `nixling.vms.<vm>.guestConfigFile` (a contained guest-editable OS layer) + the `nixling config` sync/diff/approve workflow. Containment is a fail-closed host-eval namespace policy lint (no `microvm.*`/`nixling.*` from the guest file, detected by definition-existence over the real NixOS module set so imports/`builtins.toFile`/`_file`-spoofing are caught — a best-effort lint, not a sound sandbox: an approved file's eval-time host reads and `mkIf`-on-divergent-context cases are bounded by operator review/approve, with a sound restricted-evaluator as deferred future work); the canonical config stays host-side; the transport is a bounded host-initiated SSH copy reusing the per-VM key (no virtiofs, no new socket); review/approve is host-operator-only and never auto-writes the config tree; guest-built store paths are never trusted. **Superseded transport (W16):** the SSH `cat` copy described here was replaced by the typed bounded `ReadGuestFile` read over the authenticated guest-control vsock (ADR 0029); the rest of the containment/review model still holds. |
| [0025. Host-jailed Wayland filter proxy role](0025-wayland-proxy-host-jailed-role.md) | Accepted (v1.3-dev) | 2026-06-10 | Adds the `wayland-proxy` process role (`nixling-<vm>-wlproxy` principal, empty caps, mandatory `w1-wayland-proxy` seccomp, no PipeWire/Pulse access, dedicated `/run/nixling-wlproxy/<vm>` runtime dir, host compositor socket bind-mounted at in-jail upstream path). Extends the nixlingd pidfd watchdog so unexpected proxy death triggers `StopRunnerRequested { runner_role: Gpu }`. GPU runners no longer hold direct access to the real host compositor socket once Wave 2 integration is complete. |

| [0027. Hardlink-backed store-view live pool](0027-store-view-hardlink-live-pool.md) | Draft | 2026-06-09 | Makes `store-view/live` the canonical hardlink-backed per-VM lower store, with Rust broker `StoreSync` as the sole writer and metadata-only generations. |
| [0028. Guest control plane over virtio-vsock](0028-guest-control-plane-over-vsock.md) | Accepted | 2026-06-08 | Selects ttRPC/protobuf for guest-control APIs and Kata-style chunked stdio RPCs for Docker-like exec I/O, with a representative static/no-unsafe guest dependency proof, LoadCredential token auth, and SSH compatibility only for existing SSH-backed commands on old running VMs; production guest binaries remain an implementation gate. Supporting evidence: [guest-control feasibility dossier](guest-control-feasibility-dossier.md). **Shipped (W16):** guest-control plane landed (`ReadGuestFile` config sync + guest exec/konsole over vsock, no SSH); the SSH compatibility path for old running VMs is not yet wired and `config sync` fails closed there. |
| [0029. Migrate framework SSH operations to typed guest-control RPCs](0029-framework-ssh-to-typed-guest-rpc.md) | Accepted | 2026-06-09 | Migrates framework readiness (raw TCP-22 → fail-closed authenticated guest-control Health) and `config sync` (SSH `cat` → typed bounded `ReadGuestFile` over the authenticated vsock channel) off SSH, adds the negotiated `ReadGuestFile` guest capability, deprecates `vm konsole`, and confines remaining SSH to delimited operator conveniences guarded by a fail-closed source allowlist. **Shipped (W16):** readiness (`guest-control-health`) and `config sync` (`ReadGuestFile`) migrated off SSH; `vm konsole` now runs over guest-control via the shipped admin-only `vm exec`; the SSH compatibility transport for old generations is not yet wired (`config sync` fails closed). |
| [0030. Guest exec runs as the workload user in a PAM login session](0030-guest-exec-as-workload-user.md) | Accepted | 2026-06-13 | Replaces root-only guest exec with the host-fixed per-VM workload user (`ssh.user`, never root) in a real PAM login session (`systemd-run --uid=<uid> -p PAMName=login` through the user login shell), reproducing the retired SSH `vm konsole` environment (`XDG_RUNTIME_DIR` + `WAYLAND_DISPLAY`). Guestd ignores the wire `user` field, names each transient unit so teardown SIGKILLs the workload cgroup after a local kill, removes `guest.exec.allowRoot`/`guest.exec.users` (tombstoned) + the `nixling-userd` stub + `vm konsole`. |
| [0031. Bare commands and detached workload-user exec](0031-bare-command-and-detached-exec.md) | Accepted (Unreleased) | 2026-06-15 | Allows bare command names behind the workload user's login shell, maps invalid program names to the usage-class `guest-control-invalid-program` error, enables non-interactive detached exec, and adds VM-first detached-management verbs with bounded per-stream retained-log offsets. |
| [0032. nixling v2 constellation control plane](0032-nixling-v2-constellation-control-plane.md) | Accepted | 2026-06-14 | Evolves `nixlingd` into a transport-neutral constellation daemon whose peer protocol connects local hosts, gateway guests, remote full hosts, cloud control-plane nodes, and provider adapters. |
