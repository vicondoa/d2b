# `closures/<vm>.json` schema reference

`closures/<vm>.json` is the private per-VM closure artifact. It records the evaluated toplevel, the closure paths required by the guest and sidecars, declared-runner parity data, and generation metadata consumed by the future daemon runner.

Producer: `nixos-modules/manifest-closures.nix` emits this artifact; `packages/nixling-core` parses it.

Schema: [`closures.json`](./closures.json) (forward reference; generated by W1 s1 with `cargo xtask gen-schemas`).

## Fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `schemaVersion` | string | yes | Artifact schema version. W1 emits `v1`. |
| `vmName` | string | yes | VM name matching `vms.json` and the enclosing `closures/<vm>.json` path. |
| `toplevel` | string | yes | Guest NixOS system toplevel store path. |
| `closurePaths` | array | yes | Complete set of store paths that must be present in the per-VM store view. |
| `declaredRunner` | object | yes | microvm.nix declared-runner path and extracted runner-shape metadata. |
| `runnerParity` | object | yes | Parity oracle data anchored to the W0b [runner-shape audit](../../runner-shape-audit.md). |
| `generation` | object | yes | Generation metadata: revision, build system, source identity, and content hashes. |

## `closurePaths`

`closurePaths` is a deterministic list. It includes the guest toplevel,
runner inputs, kernel/initrd paths, virtiofs/share dependencies, and any
feature sidecar payloads required by the VM. It does not carry secret
material or host-private key paths.

## `declaredRunner`

| Field | Description |
| --- | --- |
| `path` | Store path to the evaluated microvm.nix declared runner. |
| `hypervisor` | Declared hypervisor value; W1/W4 target Cloud Hypervisor. |
| `tapInterfaces` | TAP interface metadata extracted for parity, not as authority over host reconcile. |
| `vsockCid` | Declared CID when present; the daemon may diverge only under the documented ADR 0004 rules. |
| `argvSnapshotHash` | Hash of the declared runner argv snapshot used as parity evidence. |

## `runnerParity`

The parity data records the fields that the daemon-owned Cloud
Hypervisor argv generator must compare against: binary path, kernel,
initrd, cmdline, memory, CPUs, console/serial policy, vsock, virtiofs
shares, API socket, network devices, and feature extra args. The source
for the expected shape is
[`docs/reference/runner-shape-audit.md`](../../runner-shape-audit.md).

## `generation`

| Field | Description |
| --- | --- |
| `generatedAt` | Deterministic generation metadata string. |
| `sourceRevision` | Source revision when available. |
| `nixSystem` | Evaluation system. |
| `bundleVersion` | Bundle version this closure artifact belongs to. |
| `hashes` | Hashes for the artifact and selected closure partitions. |
