# `host.json` schema reference

`host.json` is the private host-intent artifact. It records the exact host resources and reconciliation policy that the daemon and privileged broker must enforce before any VM process starts.

Producer: `nixos-modules/manifest-host.nix` emits this artifact; `packages/nixling-core` parses it.

Schema: [`host.json`](./host.json) (forward reference; generated by W1 s1 with `cargo xtask gen-schemas`).

## Top-level fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `schemaVersion` | string | yes | Artifact schema version. W1 emits `v1`. |
| `supportTier` | object | yes | Host support tier, platform, kernel floor, and detected feature requirements from [ADR 0008](../../../adr/0008-supported-platforms-and-rejected-targets.md). |
| `ifNamePolicy` | object | yes | `IfName` validation and derivation rules. |
| `environments` | array | yes | Per-env network intent: bridges, TAPs, MTU/MSS/east-west policy, route and hosts entries. |
| `nftables` | object | yes | Exact `inet nixling` table hooks, priorities, chains, and rule ownership policy. |
| `networkManager` | object | yes | Unmanaged configuration materialization rules. |
| `hostsFile` | object | yes | `/etc/hosts` marked-block ownership policy. |
| `ipv6Off` | object | yes | Per-link IPv6 disablement sysctls. |
| `kernelModules` | array | yes | Canonical W1 kernel-module matrix. |
| `fdOwnership` | array | yes | Broker-opened privileged resource ownership and handoff table. |
| `cloudHypervisorCapabilities` | array | yes | Backend capability matrix anchored to [ADR 0004](../../../adr/0004-cloud-hypervisor-runner-shape.md). |

## `IfName` rules

Every host-visible interface name is a typed `IfName`:

| Rule | Value |
| --- | --- |
| Maximum length | 15 bytes (`IFNAMSIZ - 1`) |
| Character set | ASCII `[A-Za-z0-9_-]+` |
| Derivation | nixling-owned prefix plus short hash-derived component |
| Truncation | Forbidden; generation must fail instead |
| Lookup metadata | User-facing env/VM names remain available in `host.json` for operators |

## Bridge-port flags

| TAP role | `isolated` | `hairpin` | `learning` | `unicast_flood` | `multicast_flood` | `neigh_suppress` | `guard` | `root_block` | `fast_leave` |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| `netVmLan` | false | false | true | true | false | false | true | true | true |
| `workloadLan` default | true | false | true | true | false | false | true | true | true |
| `workloadLan` with east-west opt-in | false | false | true | true | false | false | true | true | true |
| `uplinkPointToPoint` | false | false | true | true | false | false | true | true | true |

A workload LAN TAP may be unisolated only when both
`lan.allowEastWest = true` for the env and host-wide
`site.allowUnsafeEastWest = true` are present in the trusted bundle.

## Environment network fields

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | User-facing env name. |
| `lanBridge` | object | LAN bridge `IfName`, address, STP-off, multicast-snooping-off, and carrier policy. |
| `uplinkBridge` | object | Host-to-net-VM /30 bridge `IfName` and address policy. |
| `taps` | array | TAP name, MAC, bridge, VM owner, role, and port flags. |
| `mtu` | integer | Per-env MTU threaded to bridge, TAP, guest NIC, and net VM policy. |
| `mssClamp` | integer or null | Per-env TCP MSS clamp value applied by the net VM. |
| `lan.allowEastWest` | boolean | Env-local east-west opt-in. |
| `site.allowUnsafeEastWest` | boolean | Host-wide opt-in required before workload LAN ports are unisolated. |
| `net_vm_forward_blocklist` | array | Env blocklist plus host LAN CIDRs; loaded before broad allow rules. |

## `nftables`

The table name is fixed: `inet nixling`. The hook layout is also fixed
and is duplicated in the ADR 0005 addendum.

| Chain | Type | Hook | Priority | Policy | Purpose |
| --- | --- | --- | --- | --- | --- |
| `nl_ingress_accept` | filter | `input` | `-300` | `accept` | Early ACCEPT carve-outs for daemon-owned host endpoints. |
| `nl_forward_accept` | filter | `forward` | `-300` | `accept` | Early ACCEPT carve-outs for net-VM forwarding, DHCP/DNS, and USBIP flows. |
| `nl_egress_accept` | filter | `output` | `-300` | `accept` | Early ACCEPT carve-outs for host-originated nixling control-plane flows. |
| `nl_ingress_drop` | filter | `input` | `300` | `accept` | Late default-drop and drift guards for marked nixling inputs. |
| `nl_forward_drop` | filter | `forward` | `300` | `accept` | Late default-drop for cross-env, blocked CIDR, and non-carved forwarding. |
| `nl_egress_drop` | filter | `output` | `300` | `accept` | Late default-drop for marked nixling egress that must not escape policy. |

Priority `-300` runs before standard nft filter chains at priority `0`;
priority `300` runs after them. Nixling never flushes foreign tables.

## NetworkManager materialization

| Field | Value |
| --- | --- |
| `path` | `/etc/NetworkManager/conf.d/00-nixling-unmanaged.conf` |
| `match` | All nixling bridge and TAP `IfName` values from this artifact |
| `contents` | A generated `keyfile.unmanaged-devices` list matching interface names |
| `owner` / `mode` | root-owned, non-world-writable |
| `reload` | Reload NetworkManager after content changes when NetworkManager is active |
| `drift` | Replace only the nixling-managed file; do not edit foreign files |

## `/etc/hosts` marked block

Nixling owns only lines between deterministic begin/end sentinels. Host
prepare replaces that block atomically, preserves all foreign lines, and
fails closed if the sentinels are malformed or nested.

## `ipv6Off`

For every nixling bridge and TAP, the broker writes and verifies these
per-link sysctls before link-up:

| Sysctl suffix | Required value |
| --- | --- |
| `disable_ipv6` | `1` |
| `accept_ra` | `0` |
| `autoconf` | `0` |
| `addr_gen_mode` | deterministic non-generating value |

## `kernelModules`

| Module entry | Required posture | Source feature | Load policy / guards |
| --- | --- | --- | --- |
| `kvm` | required | Cloud Hypervisor | Must be built in or loaded. |
| `kvm_intel` or `kvm_amd` | required by CPU vendor | Cloud Hypervisor | Exactly the vendor-appropriate accelerator is required. |
| `tun` | required | TAP devices | Broker may request load unless `kernel.modules_disabled=1`. |
| `vhost_net` | required when vhost-net handoff is enabled | TAP acceleration | Broker-opened fd; no jail-visible node by default. |
| `fuse` | required | virtiofsd/store views | Required for virtiofsd roles. |
| `nf_tables` | required | nftables | Includes needed inet/ip/ip6 families. |
| `bridge` | required | Linux bridges | Required for env LAN/uplink bridges. |
| `br_netfilter` | optional/fail-closed when present | bridge filtering interaction | If loaded, `bridge-nf-call-{ip,ip6,arp}tables` must be `0`. |
| `vhost_vsock` | deferred | future kernel vsock backend | Reserved, no jail-visible row in W1. |
| GPU/vendor modules | optional by VM feature | graphics | Required only for graphics roles that declare the feature. |
| `usbip_host` | optional by VM feature | USBIP | Gated by v0.4.0 USBIP rules. |
| `vfio` | reserved/deferred | future passthrough | Not required in W1 unless a later role enables it. |
| `userfaultfd` | syscall role, not module row | future migration/snapshot | Reserved as role metadata, blocked by default seccomp. |

## `fdOwnership`

| Resource | Opener | Transport | Long-lived payload access |
| --- | --- | --- | --- |
| `/dev/kvm` | broker | `SCM_RIGHTS` fd | No bind-mounted device node unless a later ADR requires it. |
| TAP fd | broker | `SCM_RIGHTS` preferred | No retained `CAP_NET_ADMIN`; persistent `TUNSETOWNER` fallback only. |
| `/dev/vhost-net` | broker | `SCM_RIGHTS` with TAP fd | No jail-visible device node by default. |
| cgroup dirfd | broker | `SCM_RIGHTS` or delegated path | Delegated nixling subtree only. |
| `/dev/vhost-vsock` | deferred | none in W1 | No jail-visible node and no broker handoff in W1. |

## `cloudHypervisorCapabilities`

| Capability/backend | W1 posture | Implications |
| --- | --- | --- |
| `headless` Cloud Hypervisor | supported target | CH argv parity with declared runner plus daemon-owned API socket. |
| `graphics` Cloud Hypervisor | supported shape | GPU/video sidecars split from CH role. |
| `audio` Cloud Hypervisor | supported shape | Audio sidecar has separate uid/gid/cgroup/profile. |
| `tpm` Cloud Hypervisor | supported shape | `swtpm` is a separate role with pre-start flush metadata. |
| `vsock` Cloud Hypervisor | supported shape | CH Unix-socket-backed vsock only; kernel `vhost_vsock` deferred. |
| `firecracker` | deferred non-goal | Requires later ADR and parity review. |
| `crosvmAsFullVmm` | deferred non-goal | Crosvm may be a helper, not the primary W1 VMM. |
