# CLI JSON output schemas

**Diataxis category:** reference.

This directory pairs generated JSON Schema files with hand-written prose
that explains field semantics, ordering, null-vs-omitted behavior, and
stability promises for the **rust-native** read-only commands.

- `*.schema.json` files are generated by `cargo xtask gen-cli-schemas`.
- `*.md` files are maintained by hand.
- `tests/cli-json-drift.sh` is the schema/output drift gate.

## Coverage

| Command | Schema | Prose companion |
| --- | --- | --- |
| `list` | `list.schema.json` | [list.md](./list.md) |
| `status` | `status.schema.json` | [status.md](./status.md) |
| `audit` | `audit.schema.json` | [audit.md](./audit.md) |
| `host check` | `host-check.schema.json` | [host-check.md](./host-check.md) |
| `host doctor` | `host-doctor.schema.json` | [host-doctor.md](./host-doctor.md) |
| `auth status` | `auth-status.schema.json` | [auth-status.md](./auth-status.md) |

The legacy bash compatibility commands keep their existing JSON
contracts, but only the rust-native read-only set gets a dedicated
schema companion in this directory.
